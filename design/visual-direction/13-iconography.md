# PAYLIO VISUAL DIRECTION SHEET 13

# Iconography & Symbol Language

### Quiet Communication

---

## Purpose

Icons inside Paylio are not decorations.

They are signals.

Each one should communicate instantly.

Without captions.

Without explanation.

Without effort.

The iconographic system should feel like a designed language.

Consistent.

Precise.

Purposeful.

---

## Emotional Objective

Icons should create:

Clarity.

Confidence.

Familiarity.

Trust.

Ease.

Users should never pause to interpret an icon.

Understanding should arrive automatically.

---

## Story

Imagine the wayfinding system inside a world-class international airport.

No words are required in the secure zone.

Symbols alone guide millions of people every day.

Each symbol is simple.

Each symbol is universal.

Each symbol was designed with extraordinary care so that no care is required to use it.

That level of intentionality governs Paylio's iconographic language.

---

## Design Philosophy

Icons should communicate function.

Not personality.

Not decoration.

Not brand enthusiasm.

Every icon earns its presence by being genuinely necessary.

If a label is sufficient,

the icon should be questioned.

If the icon removes the need for a label,

it is doing its job.

---

## Visual Character

Rounded stroke caps.

Consistent stroke weight.

Two pixels at default scale.

Proportional to the container.

Simple geometry.

No filled shapes masquerading as line icons.

No line icons pretending to be filled.

Consistent.

Honest.

Precise.

---

## Scale

Every icon should be designed on a 24×24 pixel grid.

Deployed at 18 pixels for navigation.

Deployed at 20 pixels for list items.

Deployed at 22 pixels for quick actions.

Deployed at 24 pixels for primary controls.

Never scaled beyond 28 pixels without redesigning.

Precision at every size.

---

## Weight

Stroke weight remains consistent throughout.

1.75 pixels at 24×24.

Neither too thin to disappear.

Nor too thick to feel clumsy.

The weight should feel confident.

Not aggressive.

---

## Icon Categories

### Navigation

Home.

Transfer.

Insights.

Cards.

Security.

Five icons.

Used hundreds of times.

Each should be immediately recognizable in silhouette alone.

---

### Actions

Send.

Receive.

Add.

History.

Freeze.

Reveal.

Hide.

Edit.

Delete.

Each action icon should feel like a verb.

Precise.

Unambiguous.

---

### Status

Success.

Processing.

Failed.

Pending.

Warning.

Status icons communicate state.

Never opinion.

---

### Financial

Balance.

Transaction.

Card.

Spending.

Limit.

Financial icons should feel precise.

Like instrument dials.

Not like banking illustrations.

---

## Color

Icons inherit their context.

Active icons receive teal.

Inactive icons receive thirty-five percent white opacity.

Positive state icons receive emerald.

Caution icons receive amber.

Critical icons receive restrained red.

Icons should never display their own independent color.

Color communicates state.

Not style.

---

## Motion

Icons respond to interaction.

Touch creates immediate gentle response.

Active state transitions smoothly.

Navigation icons settle into their active state.

Not snap.

Not bounce.

Settle.

Like a precision instrument finding its resting position.

---

## Consistency

Every icon must feel designed by the same hand.

Same geometry.

Same weight.

Same radius.

Same attention.

Inconsistency destroys trust.

A mismatched icon communicates carelessness.

Users notice this subconsciously.

Even when they cannot name it.

---

## What Icons Should Never Do

Communicate brand enthusiasm.

Illustrate concepts.

Duplicate text labels unnecessarily.

Appear decorative.

Draw attention to themselves.

Compete with content.

The best icon is the one users never consciously notice.

---

## Cinematography

Imagine the icons photographed as precision instruments.

Extreme macro.

Perfect lighting.

Shallow depth.

Each stroke revealing its quality.

Every curve communicating craftsmanship.

Icons should feel engraved.

Not printed.

---

## Things to Avoid

Do not mix icon styles.

Do not use filled and outlined icons together.

Do not use emoji.

Do not use illustration-style icons.

Do not use icons from public libraries without refinement.

Do not use icons as decoration.

Do not create icons without clear function.

Do not vary stroke weight across the system.

---

## Quality Standard

Every icon should feel like it could appear on a physical device.

Etched.

Machined.

Engraved.

Premium hardware uses the same philosophy.

Small.

Precise.

Universally understood.

That is the Paylio icon standard.

## Design Principle for Claude Code

An icon that requires explanation has already failed.

Design every icon to be understood in one glance.

Test it in silhouette.

Test it at minimum size.

Test it against its context.

If it passes every test without any supporting label,

it belongs in Paylio.
