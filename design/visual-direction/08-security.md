# PAYLIO VISUAL DIRECTION SHEET 08

# Security

### Trust Made Visible

---

## Purpose

The Security experience is where Paylio earns trust.

This page is not about passwords.

It is not about PINs.

It is not about biometrics.

It is about giving users complete confidence that their financial world belongs only to them.

Security should feel calm.

Invisible when everything is normal.

Powerful when needed.

---

## Emotional Objective

The page should create these emotions.

Safety.

Ownership.

Confidence.

Privacy.

Calm.

Protection.

Users should leave feeling:

"My money is protected."

Not

"My account is locked down."

There is an important emotional difference.

---

## Story

Imagine entering the private vault of a modern financial institution.

The room is quiet.

Soft architectural lighting surrounds carefully crafted materials.

Nothing feels intimidating.

Every security system exists,

yet none of them demand attention.

You simply know you are protected.

That is the feeling Paylio should create.

---

## Design Philosophy

Security should never create fear.

It should remove fear.

The interface should communicate certainty.

Every setting should be understandable.

Every action should have immediate feedback.

Nothing should feel technical.

Nothing should require explanation.

---

## Information Hierarchy

User Identity.

↓

PIN Protection.

↓

Biometric Authentication.

↓

Security Alerts.

↓

Advanced Controls.

Everything should feel naturally organised.

Users should instantly understand what protects them.

---

## Identity

The profile area should feel personal.

Beautiful typography.

Elegant avatar.

Editable display name.

Soft lighting.

The user should feel recognised.

Not processed.

---

## PIN Experience

Entering a PIN should feel precise.

Each number responds with subtle tactile animation.

Digits softly illuminate.

Transitions remain smooth.

The keypad feels premium.

Never mechanical.

When changing the PIN,

the experience should feel like confirming ownership,

not completing a form.

---

## Biometric Authentication

Face ID and fingerprint should become beautiful moments.

The scanner animation should feel elegant.

Soft concentric light.

Gentle teal illumination.

Natural breathing motion.

No aggressive scanning effects.

No science-fiction visuals.

Biometrics should feel effortless.

Almost invisible.

---

## Security Toggles

Every switch should communicate confidence.

Soft transitions.

Immediate response.

Elegant animation.

Settings quietly update.

No loading.

No interruptions.

The interface should always reassure the user that changes have been applied.

---

## Authentication Experience

When Paylio launches,

the lock screen should feel premium.

Not restrictive.

The background remains beautifully blurred.

Ambient light softly moves.

The balance remains hidden.

The user calmly authenticates.

Within seconds,

the interface opens naturally.

Like entering a beautifully designed private office.

---

## Privacy

Privacy should be beautiful.

Sensitive information fades elegantly.

Balance visibility toggles feel immediate.

Card numbers reveal themselves only when requested.

Everything communicates respect.

The interface never exposes information unnecessarily.

---

## Lighting

Lighting represents trust.

Deep navy surroundings.

Soft teal illumination.

Gentle white highlights.

Minimal violet reflections.

No dramatic contrast.

The lighting should feel reassuring.

Not theatrical.

---

## Materials

Dark frosted glass.

Matte ceramic.

Brushed aluminium.

Premium OLED lighting.

Beautiful reflections.

Soft transparency.

Nothing cheap.

Nothing artificial.

---

## Motion Philosophy

Every security interaction should be calm.

PIN digits appear naturally.

Biometric scanning gently pulses.

Confirmation softly fades.

Successful authentication creates a subtle wave of light.

The interface welcomes the user.

Motion should reduce anxiety.

Never increase it.

---

## Atmosphere

Imagine standing inside a luxury private bank after closing hours.

Everything is silent.

Every material has purpose.

Every light has intention.

The room feels secure,

yet welcoming.

That atmosphere defines Paylio Security.

---

## Color System

Deep Navy.

Midnight Blue.

Soft Teal.

Muted Violet.

Pure White.

Emerald for successful verification.

Amber for caution.

Restrained Red for genuine security problems.

Every color communicates meaning.

Never emotion.

---

## Typography

Typography remains calm.

Simple language.

Short instructions.

Clear hierarchy.

Large spacing.

Nothing crowded.

Users should never need to reread instructions.

---

## Cinematography

Imagine filming security for Apple Wallet.

Slow macro camera movement.

Soft reflections.

Beautiful depth.

Elegant scanner illumination.

Gentle breathing light.

The user should feel invited,

not challenged.

Every frame communicates trust.

---

## Things to Avoid

Do not use glowing hacker graphics.

Do not use padlock illustrations.

Do not use shield icons everywhere.

Do not create aggressive fingerprint animations.

Do not imitate cyber-security dashboards.

Do not use alarming colors.

Do not create unnecessary warnings.

Security should feel sophisticated.

Not frightening.

---

## Quality Standard

This page should redefine how security feels.

Instead of reminding users about danger,

it reminds them about protection.

Every interaction should quietly reinforce trust.

Every animation should communicate reassurance.

Every detail should feel worthy of a premium financial institution.

## Design Principle for Claude Code

Security is not a barrier.

It is a promise.

Every interaction should quietly tell the user:

**"You are the only person who belongs here."**

If the dashboard is where users feel financially confident, the Security Center is where they feel emotionally safe. Those two feelings together define trust, and trust is the foundation of the Paylio brand.
